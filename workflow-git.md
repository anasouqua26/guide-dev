Branch : On crée une branche isolée (ex: feature/nom) pour travailler sur une nouvelle fonctionnalité sans casser le code principal (main).

Commit : On enregistre localement nos modifications avec des messages clairs. Cela permet de garder un historique des étapes de développement.

Push : On envoie nos commits locaux vers le dépôt distant (GitHub/GitLab) pour que les autres puissent voir le travail.

Pull Request (PR) : On demande officiellement la fusion de notre branche vers la branche principale. C'est le moment de la revue de code par les collègues.

Merge : Une fois la PR validée, on fusionne les modifications dans la branche main. La fonctionnalité est alors officiellement intégrée.
